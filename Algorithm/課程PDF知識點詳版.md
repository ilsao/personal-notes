# 課程 PDF 知識點詳版

## FromCtoCPP.pdf
- 電腦系統基本觀念：硬體本身不會「自動解題」，需要作業系統與應用程式協同運作。
- 程式執行模型：程式在記憶體中以 0/1 指令串流表示，CPU 透過 fetch-decode-execute 週期執行。
- Von Neumann 架構：程式與資料儲存在記憶體，透過程式計數器依序取指令。
- 程式語言定位：C 側重結構化；C++ 以物件導向為核心並引入 template/STL。
- 結構化程式設計重點：把任務切分成可管理的小函式、強調可維護性。
- 物件導向程式設計重點：先設計資料物件，再設計作用於資料的方法，並透過介面使用與資料隱藏。
- 以 Queue 作為 ADT 範例：定義資料與操作，不暴露內部實作細節。
- Queue 的基本操作語意：入隊（rear 端加入）、出隊（front 端移除）、先進先出。
- Queue 的陣列實作概念：以 `front/rear` 指標追蹤狀態，對應 `Inqueue/Dequeue`。
- C 與 C++ 在泛型支援上的差異：C++ 可用 template 讓型別參數化，減少重複實作。
- 註解語法：C 使用 `/* */`；C++ 同時支援 `/* */` 與 `//`。
- 標頭檔與前處理：`#include <...>` 與 `#include "..."` 的用途與替換行為。
- 基本型別：`char/int/float/double/void`，以及 C++ 新增 `bool`。
- 布林與數值轉換：`true/false` 可被轉型，需理解語意與風險。
- 混合型別運算規則：運算前先做型別提升（通常往較大表示範圍），指定時再轉回左值型別。
- 型別轉換風險：C 允許較自由轉換，雖方便但容易隱藏錯誤；C++ 常給警告。
- 顯式轉型建議：關鍵處手動 cast 以降低隱式轉換造成的 bug。
- 整數除法陷阱：`int/int` 仍為整數結果，需先提升型別才能得到浮點結果。
- `enum`（列舉型別）定義與宣告方式。
- `struct` 觀念：可視為簡化 class（預設成員公開），並可搭配存取控制。
- 變數作用域：全域、區域、區塊作用域的可見性與遮蔽現象。
- 作用域解析運算子 `::`：區分外部作用域與類別作用域成員。
- 常數定義：`#define` 與 `const` 的差異（型別檢查、除錯可見性）。
- `const` 在函式參數中的意義：保證不修改輸入資料。
- 函式原型（prototype）用途：先宣告介面，協助編譯器做參數型別檢查與轉換。
- C 與 C++ 對 prototype 的要求差異：C++ 要求更嚴格。
- `main` 標準型態：`int main()` 或 `int main(int argc, char* argv[])`。
- 命令列參數：`argc/argv` 的意義與使用方式。
- 程式回傳狀態：成功/失敗狀態碼語意。
- 參考（reference）概念：同一記憶體位置的別名機制。
- 指標（pointer）概念：儲存位址，透過解參考操作資料。
- reference 與 pointer 對照：語法、可空性、重新綁定行為差異。
- 參數傳遞：傳值 vs 傳參考，對效能與可修改性的影響。
- 以參考回傳值：可回傳既有物件別名，但需避免懸垂參考。
- `inline` 函式：以程式碼展開減少呼叫開銷，代價是可執行檔可能變大。
- macro 與 inline 比較：macro 無型別檢查，容易受括號與展開副作用影響。
- 預設參數（default arguments）：缺參時自動補值，需注意宣告規則。
- 函式多載（overloading）：同名不同參數簽章，由編譯器依呼叫解析。
- 運算子多載：讓自訂型別支援自然語法（如 `+`）。
- 動態記憶體：`new` 配置、`delete` 釋放、`new[]/delete[]` 對應陣列。
- 與 C 的 `malloc/free` 差異：C++ 具型別語意與建構/解構流程整合。
- C++ I/O 流：以 `cin/cout` 與串流運算子 `>>/<<` 進行輸入輸出。
- 串流語意：輸入流、輸出流與緩衝概念。
- 字串輸入細節：`>>` 受空白終止，需 `getline` 讀整行。
- 格式控制器（manipulator）：`dec/hex/oct/endl/ws/flush/setw/setprecision/setfill`。
- namespace 目的：大型專案避免名稱衝突。
- `using namespace` 的便利與命名污染取捨。

## OOPClass.pdf
- OOP 核心三要素：封裝、繼承、多型。
- 封裝（Encapsulation）：把資料與操作綁在一起，形成抽象資料型別。
- 資料隱藏（Data Hiding）：外部透過方法存取，不直接碰內部表示。
- 繼承（Inheritance）：由既有類別衍生新類別，重用並擴充功能。
- 多型（Polymorphism）：同名介面可對不同型別有不同實作。
- 類別（class）定位：一組物件的抽象原型與生成器。
- 物件（object）定位：類別的實例，包含屬性與行為。
- ADT 例子：circle 類別、priority queue 類別。
- 成員存取語法：`obj.member` 與 `ptr->member`。
- 繼承術語：base/superclass 與 derived/subclass。
- 衍生類別能力：可新增資料/方法、可重新定義繼承方法。
- class hierarchy 觀念：容器/集合/串列/堆疊/佇列之間的繼承關係。
- 以 pen/colored pen 示範「在既有功能上擴充」。
- 以 `shape*` 指向 circle/line 示範多型呼叫 `draw()`。
- 存取修飾詞：`private/public/protected` 與預設 `private`。
- 類別設計原則：資料通常 `private`，介面方法通常 `public`。
- Stack 類別示例：私有陣列與 top，透過 `push/pop/empty` 操作。
- 類別宣告本身不配置實體資料，只有建立物件才配置。
- 建構子（constructor）角色：物件建立時自動初始化。
- 建構子特性：與類別同名、無回傳型別、可多載、可有預設參數。
- 建構子不可繼承，且通常應設為 `public`。
- 為何需要建構子：避免忘記手動初始化導致未定義狀態。
- 預設建構子與參數化建構子：不同初始化情境。
- 成員初始化需求：資料成員應由建構流程設定。
- 無使用者自訂建構子時，編譯器可產生預設建構子（受條件限制）。
- 拷貝建構子（copy constructor）：以同類別參考初始化新物件。
- 轉換建構子（conversion constructor）：單參數建構子可做型別轉換。
- 建構子任務：初始化 +（必要時）動態配置資源。
- 解構子（destructor）任務：物件銷毀時釋放資源。
- 解構子特性：`~ClassName()`、無參數、無回傳、離開作用域自動呼叫。
- ZipCode/Battery 範例：展示多建構子與資源管理需求。

## InfixPostfixExpression.pdf
- 運算式型態：中序（infix）與後序（postfix）表示法。
- 基本元素：operand（運算元）與 operator（運算子）。
- 運算子分類：二元運算子與一元運算子。
- 優先權概念：中序需靠優先權（及括號）決定實際運算順序。
- 中序到後序轉換觀察：運算元順序通常不變，運算子順序會依優先權重排。
- 轉換工具：使用 stack 暫存運算子以處理優先權逆轉。
- 基本演算法流程：
- 從左到右掃描 token。
- 遇運算元直接輸出。
- 遇運算子時，與 stack top 比較優先權後決定 push 或 pop。
- 掃描完成後，將 stack 內剩餘運算子依序輸出。
- 括號處理重點：括號改變語意，需設計 in-stack/out-stack priority。
- `(` 必須可被推入並隔離括號內運算，遇 `)` 需彈出直到匹配 `(`。
- 範例推導：`a+b*c`、`a*b/c`、`a+b-c*d/e+f`、`x+(a+b-c)*d/(e+f)`。
- 課程銜接：後續主題導向 Linked List、Tree、Binary Search Tree。

## RBtree, RBtreeBook, AVLtree-20260324/RBT.pdf
- 紅黑樹是具顏色標記的高度平衡 BST，每個節點帶 `color/key/left/right/p`。
- 空子節點以黑色 `Nil`（或單一 `T.nil`）表示，簡化邊界條件處理。
- 紅黑樹五大性質：
- 節點非紅即黑。
- 根為黑。
- 葉（Nil）為黑。
- 紅節點的子節點必為黑。
- 任一節點到其所有後代葉的路徑黑節點數相同。
- black-height 概念：從某節點到葉的黑節點數（不含自身）。
- 高度上界結論：高度受 black-height 約束，操作可維持 `O(log n)`。
- 可在 `O(log n)` 完成搜尋、最小/最大、前驅/後繼等。
- 旋轉操作：左旋與右旋，局部調整結構以修復平衡。
- BST 刪除回顧：刪除 0/1 子節點、或以後繼替代 2 子節點情況。
- RB 插入流程：
- 先按 BST 插入，新增節點著紅。
- 若父黑則結束；若父紅需修復連續紅問題。
- 依叔叔顏色與相對位置分 case（含旋轉與重著色）。
- RB 刪除流程：
- 追蹤被刪或被搬移節點顏色。
- 若刪除黑節點可能出現 double-black，需要 fixup。
- fixup 依兄弟與兄弟子女顏色分 case，透過旋轉與重著色恢復性質。

## RBtree, RBtreeBook, AVLtree-20260324/rbtreeBook.pdf
- 章節主軸：以嚴謹性質、偽碼、正確性不變量、案例分析完成紅黑樹插刪設計。
- 13.1 性質與理論基礎：
- 紅黑樹為 BST + 顏色位元。
- 透過路徑顏色約束，保證最長路徑不超過最短路徑兩倍。
- 單一 sentinel `T.nil` 代表所有 NIL 葉與根父節點，降低實作複雜度。
- black-height（`bh(x)`）定義與可良定性（由性質 5 保證）。
- Lemma 13.1：`n` 個內部節點的紅黑樹高度至多 `2lg(n+1)`。
- 推論：`SEARCH/MINIMUM/MAXIMUM/SUCCESSOR/PREDECESSOR` 皆為 `O(log n)`。
- 13.2 Rotation：
- LEFT-ROTATE/RIGHT-ROTATE 只改常數個指標，時間 `O(1)`。
- 旋轉保留 BST 中序順序與鍵值關係。
- 偽碼細節：父子指標更新順序、根替換、子樹接回位置。
- 13.3 Insertion：
- `RB-INSERT`：沿 BST 路徑插入後，設 `z.left=z.right=T.nil` 並令 `z` 為紅。
- 為何先設紅：減少破壞 black-height，修正成本較低。
- 可能違反性質只剩 2（根黑）與 4（紅紅相鄰）。
- `RB-INSERT-FIXUP` 三案例（另有左右對稱）：
- Case 1：叔叔紅，父叔轉黑、祖父轉紅，`z` 上移兩層。
- Case 2：叔叔黑且 `z` 內側子，先旋成 Case 3 形態。
- Case 3：叔叔黑且 `z` 外側子，重著色 + 單旋，結束。
- loop invariant 重點：每輪最多保留單一違規（性質 2 或 4）。
- 複雜度：總時間 `O(log n)`，最多 2 次旋轉。
- 13.4 Deletion：
- `RB-TRANSPLANT`：可無條件設定 `v.p`（即使 `v=T.nil`）。
- `RB-DELETE` 追蹤 `y-original-color` 與 `x`（搬到原位置的節點）。
- 若移除/搬移的是黑節點，可能破壞性質 1/2/4/5，需 `RB-DELETE-FIXUP`。
- extra-black 模型：把缺失黑色暫記在 `x`（doubly-black 或 red-and-black）。
- `RB-DELETE-FIXUP` 四案例（另有左右對稱）：
- Case 1：兄弟紅，先轉成兄弟黑的情境。
- Case 2：兄弟黑且兩子黑，將額外黑往上推。
- Case 3：兄弟黑、近子紅遠子黑，先轉成 Case 4。
- Case 4：兄弟黑且遠子紅，旋轉與重著色後移除 extra black。
- 複雜度：`O(log n)`，最多 3 次旋轉。
- 正確性重點：每個 case 都維持黑高一致（性質 5），並最終恢復全部紅黑性質。

## RBtree, RBtreeBook, AVLtree-20260324/AVLtree.pdf
- AVL 樹定義：每個節點左右子樹皆為高度平衡，且高度差絕對值不超過 1。
- balance factor（BF）定義：`BF(T)=hL-hR`，AVL 節點 BF 僅能是 `-1/0/1`。
- 插入可能造成失衡：某些祖先節點 `|BF|>=2`。
- 修正策略聚焦於「最近失衡祖先 A」。
- 四種失衡型態與對應調整：
- LL：新節點在 A 左子樹的左側，通常做右旋。
- RR：新節點在 A 右子樹的右側，通常做左旋。
- LR：新節點在 A 左子樹的右側，先左旋子樹再右旋 A。
- RL：新節點在 A 右子樹的左側，先右旋子樹再左旋 A。
- AVL 核心目標：維持嚴格高度平衡，保障查詢/插入/刪除效率。
