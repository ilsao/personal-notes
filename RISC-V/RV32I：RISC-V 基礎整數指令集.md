
# RV32I 指令格式

下圖展示了 RV32I 基礎指令集的總攬。

![[RISC-V/pic/U2/1.png]]

下圖展示了六種基本指令格式：
- R 型指令：寄存器操作。
- l 型指令：短立即數與訪存 load 操作。
- S 型指令：訪存 store 操作。
- B 型指令：條件跳轉操作。
- U 型指令：長立即數操作。
- J 型指令：無條件跳轉操作。

![[RISC-V/pic/U2/2.png]]

注意到，圖中立即數的存儲被拆分了。這是因為最好將 `opcode, funct3, rs1, rs2` 的位置固定，使得解碼速度增快。

特別注意 B 型與 J 型指令，它們的立即數從 1 開始。這是因為 RV32I 的指令總是內存對齊的，所以最低位 (即，`imm[0]`) 總是為 0。

![[RISC-V/pic/U2/4.png]]

注意到 `pc` 並不是 RV32I 的一個標準寄存器之一。如果把 `pc` 當作寄存器，就代表所有指令都可能更改 `pc`，使得流水線預測變得困難。

# RV32I 整數計算

簡單的算術、邏輯、移位指令和其他 ISA 差不多，從寄存器讀取兩個 32 位值，並將結果寫入目的寄存器中。

同時，RV32I 還提供這些指令的立即數版本，這些立即數總是進行符號擴展。

我們可以用 `slt` 或 `sltu` 在當第一個操作數小於第二個操作數時將特定寄存器置一。

而 `lui` 可以將高 20 位載入目的寄存器的高 20 位，幫助創建較大的常量。

我們可以通過組合 `auipc` 與 `jalr` 來跳轉到任意 32 位相對地址，組合 `auipc` 與 `lw` 來訪問任意 32 位相對地址。

# RV32I 的 Load 和 Store

在加載與存儲時，有符號字節和半字先被符號擴展到 32 位再寫入目的寄存器。而無符號字節與半字被無符號擴展到 32 位再寫入目的寄存器。

注意，RV32I 唯一支持的尋址模式是基地址寄存器值 + 符號擴展 12 位立即數。(即 x86-32 的位偏移循址模式)

# RV32I 條件分支

因為 RISC-V 指令長度需要是 2 字節 (16 位) 的倍數 (兼容 16 位壓縮指令)，所以分支指令的尋址地址為 12 位立即數符號擴展左移 1 ，然後加到當前 `pc` 上。這種 `pc` 相對循址使得代碼成為 PIC，簡化鏈接器與加載器的工作。

我們可以使用條件設置與條件分支檢測溢出：

``` asm
add t0, t1, t2        # t0 = t1 + t2
slti t3, t2, 0        # t3 = t2 < 0
slti t4, t0, t1       # t4 = (t1 + t2) < t1
bne t3, t4, overflow  # if (t2 < 0 && (t1 + t2) >= t1) || (t2 >= 0 && (t1 + t2) < t1) then overflow
```

# RV32I 無條件跳轉

無條件跳轉擁有兩種功能，而且可以通過設置當前指令的下一條指令存放在哪個寄存器中實現。

當將 `PC + 4` 地址保存在返回地址寄存器 `ra` 時，可以用來實現過程調用。如果保存在 `x0` 寄存器中，因為該寄存器不可修改，可以實現無條件跳轉。

而使用 `jalr` 可以調用動態計算的函數 (地址存在寄存器)，也可以實現調用返回 (通過設置 `ra` 為源寄存器，`x0` 為目的寄存器)。 `switch` 也可以通過 `jalr` 將目的寄存器設為 `x0` 實現。
